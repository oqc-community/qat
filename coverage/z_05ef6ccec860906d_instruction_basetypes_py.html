6class PostProcessType(Enum): 

7 """`PostProcessType` is used in a `PostProcessing` instruction to determine how readouts 

8 should be processed. It describes a type of classical post-processing: 

9 

10 - `DOWN_CONVERT`: Down-converts the readout measurement, should only be used along the axis `ProcessAxis.TIME`. 

11 - `MEAN`: Takes an average over the given axis. 

12 - `LINEAR_MAP_COMPLEX_TO_REAL`: Maps the (complex) measured value onto a (real) z-projection using a supplied 

13 linear mapping. 

14 - `DISCRIMINATE`: Converts a z-projection to a classical bit by comparison with a supplied discrimination threshold. 

15 """ 

16

27class ProcessAxis(Enum): 

28 """`ProcessAxis` is used during classical post-processing of readouts. It specifies the 

29 axis which the post-processing should occur on. Often used in conjunction with the 

30 `AcquireMode` to determine the correct method for post-processing. 

31 

32 - `TIME`: Instructs the post-processing to be performed over the time-series data returned from a readout. 

33 - `SEQUENCE`: Instructs the post-processing to be performed over the shots. 

34 """ 

35 

36 TIME = "time" 

37 SEQUENCE = "sequence"

43class AcquireMode(Enum): 

44 """The `AcquireMode` is used to specify the type of acquisition at the level of the 

45 control hardware. 

46 

47 Note that different backends will only allow selected acquisition modes. 

48 

49 - `RAW`: Returns the time-series acquisition data for each shot 

50 - `SCOPE`: Returns the time-series acquisition data where each point is averaged over the number of shots 

51 - `INTEGRATOR`: Down-conversion and integration of the acquisition data has occurred on the hardware so that a 

52 single data point is returned for each shot 

53 """ 

54 

55 RAW = "raw" 

56 SCOPE = "scope"
